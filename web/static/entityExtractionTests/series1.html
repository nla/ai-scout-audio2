The prompt was adapted using trial and error from Graeme Foster's sample prompt presented as part of a recent (Nov2024) Microsoft GraphRAG presentation

Main adaptations:

  • Simplified instructions; removed duplication within instructions.
  • Removed uppercase instructions (but keep in entity/relationship examples).
  • Allowed flexibility in naming of entities and relationships. This may be controversial, but the intent here is to allow the LLM to find the best names for these, and to refine/group/organise them later using another LLM stage, given the context extracted as the "reason" value for each entity and relationship, and some target knowledge graph (wikidata?). It is likely that entity and relationship names will themselves form part of a knowledge graph, for example, "married to" and "cousin of" are subsets of "personally related to"; "worked at (some organisation)", "attended (a church or school)", "member of (some political organisation)", "member of (some artistic movement)" are subsets of (perhaps) "part of".
  • Fine-tuned one-shot example and desired response.

System prompt

Your task is to extract significant entities and their relationships from the provided text.

Entity Types:

Identify core entity types like PERSON, ORGANIZATION, PLACE, EVENT.
For novel entities, classify them based on semantic similarity to existing types or create new types as needed.

Relationships:

Identify relationships between entities based on semantic similarity, temporal proximity, or syntactic cues.
Common relationship types include: VISITED, ATTENDED, CREATED, IS_A, PART_OF, HAS_PROPERTY, OCCURS_AT, INVOLVES.

You should be flexible in identifying new relationship types as needed.

Please output the results in JSON format as lists of entities and relationships.  Each entity should show the 
entity name as found in the text, the type of entity and the reason you have extracted it.  Each relationship
should show the name and type of two entities in the relationship, the name and type of the relationship, and
the reason you have extracted it, supported by a snippet of text.
